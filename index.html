apache 加固

修改目录权限755

将所有 Web 目录设置为不可写,文件只读

httpd -V或apache2 -V可以查看到配置文件路径，

一般在/etc/apache2/apache2.conf或/etc/apache2/httpd.conf。

也可以find / -name httpd.conf或find / -name apache*.conf进行查找。

在配置文件中限制ip访问：

除了来自hacker.com域和IP地址为192.168.16.111的客户机外，允许所有客户机的访问

Order deny，allow

Deny from hacker.com

Deny from 192.168.16.111

仅允许来自网络192.168.16.0/24客户机的访问，但IP地址为192.168.16.111的客户机除外

Order deny，allow

Allow from 192.168.16.0/24

Deny from 192.168.16.111

sudo apachectl restart

PHP加固

find / -name php.ini查找配置文件路径,也可以通过phpinfo页面获取。

mysql加固

查找配置文件路径，一般/etc/my.cnf

另外可以通过mysql –verbose –help|grep -A 1 ‘Default options’命令确认mysql配置文件的加载顺序,查找配置项时应在所有这些可能的配置文件中查找。

确认mysql用户：

mysql运行账号需要给予程序所在目录的读取权限，以及data目录的读取和写入权限；不容许给予其他目录的写入和执行权限，特别是有网站的；取消mysql运行账户对于cmd，sh等一些程序的执行权限。

在my.ini（或其他一起组合的配置文件）中：

bind-address=127.0.0.1 //限制本地访问

local_infile=0 //防止LOAD DATA INFILE 或者 SELECT local_file文件读取

验证：

load data local infile ‘d:\pet.txt’ into table tmp;

show variables where variable_name=’local_infile’;

service mysql restart 重启mysql使配置生效。
